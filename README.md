## English

This repository contains a community port of **Flux Networks** for **NeoForge 26.1.2.75**.

I downloaded the original Flux Networks repository from:

https://github.com/SonarSonic/Flux-Networks

Then I updated the project on my own to make it compatible with the newer NeoForge environment.

This is not an official Flux Networks release and I am not one of the original developers.

### Important notice

This update was made with the help of artificial intelligence.

Because of that, the source code may contain mistakes, incomplete changes, compatibility problems, or unexpected bugs.

I tested some of the main features in singleplayer, but the mod may still have issues that I have not discovered yet.

I have not tested the mod on multiplayer or dedicated servers because I do not have friends to test it with :(

Use this version at your own risk and make backups of your worlds before installing it.

### Current status

* Updated for **NeoForge 26.1.2.75**
* Main Flux Networks mechanics appear to be working
* Flux Plug, Flux Point, Flux Controller, and energy networks appear to work
* Energy transfer, storage, and management appear to work
* Dependencies and integrations were adjusted for the newer NeoForge environment


### Recent fixes

* Added NeoForge energy capability registration for Flux Plug and Flux Point so modded energy pipes can connect more reliably.
* Fixed Flux Plug and Flux Point color rendering for the selected network color.
* Fixed network color refresh when selecting or editing a network, including loaded blocks already placed in the world.
* Fixed block color sync after changing dimensions or reopening a world.
* Fixed Wireless Charging inventory button texture slicing.
* Fixed configured Flux devices dropping items with their saved network, name, transfer, color, and energy data.

### Project versions

* **NeoForge:** 26.1.2.75
* **Minecraft/NeoForge base:** 26.1.2
* **Flux Networks:** 7.2.1.15
* **Java:** 25

### Reporting problems

When reporting a problem, please include:

* NeoForge version
* Installed mod list
* Latest log or crash report
* Steps required to reproduce the problem
* Whether the issue happened in singleplayer or multiplayer
